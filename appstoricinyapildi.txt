[A1] KRİTİK DÜZELTMELER (Yapılmazsa Kesin Red Gelir)
--------------------------------------------------------------

A1.1 Bundle Identifier
     Placeholder ("com.myapp") kesinlikle değiştirilmeli.
     Format: com.şirketadı.uygulamaadı
     Örnek: "com.corut.mathpulse"
     Dosya: app.json → ios.bundleIdentifier + android.package
     NOT: Bir kez yayınladıktan sonra değiştirilemez.

A1.2 Gizlilik Politikası (Privacy Policy)
     Apple zorunlu tutuyor — olmadan submit edilemez.
     İçermesi gerekenler:
     - Hangi veriler toplanıyor (veya toplanmıyor)
     - Veriler nasıl kullanılıyor
     - Üçüncü taraf paylaşımı var mı
     - İletişim bilgisi (gerçek e-posta)
     - COPPA uyumu (çocuk verisi)
     - KVKK/GDPR uyumu (AB için)
     Nerede host edilir: GitHub Pages, Notion, kendi site
     Uygulama içinden erişilebilir olmalı (Ayarlar ekranında link)

A1.3 Support URL
     Apple çalışan bir support URL istiyor.
     mailto: linki kabul etmiyor — gerçek web URL'i gerekli.
     Önerilen: GitHub repo URL'i veya basit landing page
     Uygulama içinden de erişilebilir olmalı (Ayarlar ekranında)

A1.4 Slug, Scheme, Origin Temizliği
     Development ortamından kalan ayarlar temizlenmeli:
     - slug: gerçek uygulama adı
     - scheme: gerçek uygulama adı
     - Replit/development origin'leri kaldırılmalı

A1.5 iOS Build Number
     app.json → ios.buildNumber: "1"
     Her yeni App Store submission'da artırılmalı (2, 3, 4...)
     version (1.0.0) ile buildNumber (1) farklı şeyler.

A1.6 Şifreleme Beyanı
     app.json → ios.infoPlist:
     "ITSAppUsesNonExemptEncryption": false
     Bu olmazsa App Store Connect her seferinde manuel sorar.
     Eğer özel şifreleme kullanıyorsan true yap ve belge yükle.

A1.7 İsim Çakışması Kontrolü
     App Store'da aynı isimde popüler uygulama var mı kontrol et.
     Varsa isim değiştir — yoksa:
     - Arama sonuçlarında kaybolursun
     - Apple "copycat" red'i verebilir
     - Organik trafik rakibe gider
     Araç: App Store'da manuel arama + AppTweak/ASOMobile


[A2] CRASH / BUG ÖNLEMELERİ (Guideline 2.1)
--------------------------------------------------------------

Apple başvuruların %40+'ını "App Completeness" nedeniyle reddediyor.

A2.1 ErrorBoundary
     Tüm uygulama ErrorBoundary ile sarılmalı.
     Crash yerine kullanıcıya hata ekranı gösterilmeli.
     "Tekrar Dene" butonu ile yeniden yükleme imkanı.

A2.2 AppState Listener (Arka Plan Yönetimi)
     Uygulama arka plana atılınca:
     - Aktif işlemler durdurulmalı (timer, oyun, ses)
     - State korunmalı veya güvenli şekilde sıfırlanmalı
     - Geri dönüşte crash olmamalı

A2.3 Ses Sistemi Güvenliği
     - Tüm audio işlemleri try/catch ile sarılmalı
     - staysActiveInBackground: false
     - Sound enabled/disabled değiştiğinde mevcut sesler durdurulmalı
     - Ses cleanup düzgün yapılmalı (memory leak önleme)
     - Headphone takma/çıkarma durumları test edilmeli

A2.4 console.log/warn/error Temizliği
     Production build'de gereksiz log olmamalı.
     console.warn → kaldır veya __DEV__ ile sar
     console.error → __DEV__ ile sar

A2.5 API/Network Crash Önleme
     Backend kullanılmıyorsa API kodu kaldırılmalı veya
     fallback eklenmeli. Env variable yoksa crash olmamalı.

A2.6 Test Edilmesi Gereken Senaryolar
     - İlk açılışta beyaz ekran var mı?
     - İnternet yokken crash oluyor mu?
     - Arka plan → ön plan geçişi
     - Telefon kilitle / aç
     - Silent mode açık/kapalı
     - Kulaklık tak/çıkar
     - Bildirim geldiğinde
     - Farklı iPhone boyutlarında UI taşması
     - 20+ kez aç-kapa testi

[E1] APPLE'IN EN SIK RED VERDİĞİ 8 MADDE
--------------------------------------------------------------

1. Minimum Functionality (4.2)
   Çözüm: 5+ ekran, derinlik, ilerleme, ayarlar

2. Crash/Bug (2.1)
   Çözüm: ErrorBoundary, try/catch, AppState, test

3. Gizlilik (5.1)
   Çözüm: Privacy Policy + uygulama içi link + doğru beyan

4. Metadata Uyumsuzluğu (2.3)
   Çözüm: Gerçek screenshot, doğru açıklama, çalışan linkler

5. IAP Hataları (3.1.1)
   Çözüm: Apple IAP kullan, Restore Purchases butonu ekle

6. Tasarım/HIG
   Çözüm: Safe area, okunabilir font, 44pt butonlar, animasyonlar

7. Kopya/Taklit (4.1)
   Çözüm: Özgün isim, özgün mekanik, isim çakışması kontrolü

8. iPad Uyumsuzluğu
   Çözüm: supportsTablet: false veya düzgün iPad UI
